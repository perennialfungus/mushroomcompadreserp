# Internal Beta Launch Checklist

Use this checklist before opening the MVP to Mushroom Compadres staff.

## Access and Roles

- [ ] Confirm every staff member has an invited/active user record.
- [ ] Confirm role assignments match the beta test plan: owner/admin, production/farm, packing/fulfillment, sales/wholesale, auditor.
- [ ] Confirm non-admin users cannot open admin screens.

## Operational Readiness

- [ ] Run a smoke pass for dashboard, inventory, stock counts, lots/QC, production, purchasing, Shopify, wholesale, CRM, MRP, reports, and traceability.
- [ ] Confirm offline stock count and reconnect upload behavior on at least one desktop and one mobile device.
- [ ] Confirm QR/manual scanning fallback works on staff devices.
- [ ] Confirm Shopify dashboard clearly identifies unmapped variants, locations, webhooks, inventory push, and fulfillment status.

## Feedback Loop

- [ ] Confirm the Feedback button is visible on all primary logged-in screens.
- [ ] Submit one feedback item from each staff role.
- [ ] Attach a screenshot to at least one feedback item.
- [ ] Triage feedback through new, acknowledged, planned, in progress, done, and declined.
- [ ] Export feedback as CSV and JSON for build planning.

## Release Notes

- [ ] Publish release notes for the beta version.
- [ ] Confirm logged-in non-admin users can view published release notes.
- [ ] Confirm draft release notes are visible only in admin tools.

## Data and Compliance

- [ ] Confirm feedback includes screen, workflow, module, severity, role, device, notes, and reproduction context.
- [ ] Confirm feedback triage writes audit events.
- [ ] Confirm screenshot uploads are treated as internal-only attachments.
- [ ] Confirm no public support portal or email ticketing flow is exposed.

## Launch Decision

- [ ] Owner/admin signs off on known launch risks.
- [ ] Create a build intake summary from exported feedback.
- [ ] Schedule the first beta review session.
